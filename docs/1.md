# Street Preview 生产时间轴拆分方案

## Summary

把当前“生成街道”页从“只展示最终结果”改成“生产步骤时间轴 + 最终展示输出”两段式流程。

核心原则：

- `Run Street` 仍然只跑一次完整 compose，不拆成多次生成。
- 生成完成后，基于同一批中间结果导出一组按时间累积的阶段快照。
- 主界面改为“时间轴主视图”：一个 3D 主窗口按步骤切换，同步显示该步骤的辅图和摘要。
- 最终 `Presentation Views` 保留，但只代表最终展示图，不再承担生产过程说明。

## Key Changes

### 1. 后端增加“生产步骤快照”导出

- 在 `src/roadgen3d/street_layout.py` 增加阶段快照导出层，不改 solver / retrieval 主逻辑。
- 生成顺序固定为 7 个累计阶段：
  - `road_base`
    道路、车行道、人行道、路缘、车道线。
  - `land_use_zoning`
    `road_base` + 用地 / zoning 代理体块。
  - `buildings`
    `land_use_zoning` + 周边建筑。
  - `poi_context`
    `buildings` + POI markers / exclusion rings。
  - `furniture_anchor`
    `poi_context` + POI 锚定家具。
  - `furniture_required`
    `furniture_anchor` + 非锚定但 required 的家具。
  - `furniture_optional`
    `furniture_required` + optional 家具；这一步等同最终生产态 3D 场景。
- 每个阶段都导出同坐标系、同场景范围的 3D GLB；文件放到 `out_dir/production_steps/`，文件名固定带序号前缀。
- `land_use_zoning` 和 `poi_context` 额外导出 2D companion PNG。
  - zoning 用现有 zoning preview 路径扩成 stage-aware 版本。
  - POI 用现有 POI overview 路径扩成 stage-aware 版本。
- 最终 `scene.glb`、`scene_layout.json`、`presentation_views/*.png` 继续保留不变。

### 2. 类型与 layout JSON 补齐阶段元数据

- 在 `src/roadgen3d/types.py` 新增 `ProductionStepRecord`，至少包含：
  - `step_id`
  - `index`
  - `title`
  - `glb_path`
  - `companion_path`
  - `visible_instance_ids`
  - `delta_instance_ids`
  - `counts`
- `StreetPlacement` 增加 `required: bool = False`，并序列化到 `to_dict()`。
  - 这是区分 `anchor / required / optional` 家具层级的必要字段。
- `scene_layout.json` 顶层新增 `production_steps` 列表。
- `summary` 新增最少这些字段：
  - `production_step_count`
  - `production_step_ids`
  - `final_production_step_id`
- `outputs` 新增：
  - `production_steps_dir`
  - 可选 `production_steps_manifest`（若单独写阶段清单 JSON）

### 3. 阶段场景组装规则固定

- 阶段快照不重新跑 compose，只复用一次 compose 已得到的：
  - `placement_ctx`
  - `zoning_grid`
  - `building_footprints / generated_lots / building_plans`
  - `placements`
  - `poi_ctx / exclusion_zones`
- 新增一个统一场景组装 helper，按阶段开关构建 3D 快照：
  - `include_road_base`
  - `include_zoning_proxies`
  - `include_buildings`
  - `include_poi_overlays`
  - `visible_placements`
- `poi_context` 阶段复用现有 `_add_poi_markers_and_zones()`，但仅用于阶段快照，不改最终 scene 默认行为。
- `land_use_zoning` 阶段新增轻量 3D 代理体块：
  - 用 `zoning_grid` 生成薄 slab / buffer proxy
  - 按 `lane_role` / `land_use_type` 着色
  - 不使用真实建筑 mesh
- 家具层级固定按下面规则切：
  - `anchor`: `placement_group == "street_furniture"` 且 `anchor_poi_type != ""`
  - `required`: `placement_group == "street_furniture"` 且 `required == True` 且无 anchor
  - `optional`: `placement_group == "street_furniture"` 且 `required == False` 且无 anchor

### 4. Gradio 改成“生产时间轴 + 最终展示”布局

- 在 `scripts/m1_gradio_app.py` 的 `2) 生成街道` 页，把当前单一 `Street Preview (GLB)` 改为 `Production Timeline` 主区。
- 新增这些控件：
  - `Production Step` slider
    范围固定覆盖全部 stage index
  - `Production Step Summary` textbox / markdown
  - `Production Step Preview (GLB)` Model3D
  - `Production Companion View` Image
  - `Production Step Downloads` Files
- `Run Street` 完成后：
  - 默认选中 `road_base`
  - 把 `production_steps` 写入 `gr.State`
  - slider 切换时只换已有 stage 产物，不重新生成
- 现有 `Presentation Views` 保留，但语义改成最终展示输出，只绑定最后完成场景。
- 现有 `Scene Details` / `Scene Graph` / `Spatial Distance Analysis` / `POI Analysis` 保留，默认仍读完整 `scene_layout`，不跟 timeline 同步过滤。
- `Street Preview (GLB)` 文案改名，避免和最终展示混淆。

### 5. 模板模式的降级策略固定

- 完整 7 步时间轴只对 `layout_mode="osm"` 启用。
- `template` 模式下使用简化时间轴：
  - `road_base`
  - `furniture_required`
  - `furniture_optional`
- `template` 模式不生成 `land_use_zoning / buildings / poi_context` 阶段，也不报错。
- UI 的 slider 与阶段摘要根据 `production_steps` 实际长度自适应。

## Assumptions

- 这次只优化 street preview 工作流，不改 compose 算法本身。
- 生产步骤快照是“同一场景的累计时间快照”，不是互斥分层孤岛。
- 主交互固定为“时间轴主视图”，不同时实现固定多窗口模式。
- `zoning/POI` 采用“3D 主快照 + 2D 辅图”混合展示；家具阶段只要求 3D 主快照。
- 不新增 top-level tab；所有改动都留在 `2) 生成街道` 页内。
