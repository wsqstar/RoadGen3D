## 简介
本指南面向首次接触 RoadGen3D 的用户，帮助你在约 30 分钟内完成环境搭建、依赖安装、开发环境启动与第一个场景生成的全流程。RoadGen3D 是一个"从文本描述生成 3D 城市街道场景"的神经符号系统，支持命令行与 Web 工作台两种使用方式。

## 前置条件
在开始之前，请确保系统满足以下前置条件：

- **Python 3.11+**：推荐使用 CPython 3.11 或 3.12，已在 macOS arm64 上经过充分测试
- **Git**：需要支持子模块的 Git 版本，用于克隆项目和初始化子模块
- **Node.js**：用于前端构建与开发，工作台和查看器均基于 Vite + React

## 安装步骤

### 1. 克隆项目并初始化子模块
```bash
# 克隆包含子模块的完整项目
git clone https://github.com/GIStudio/RoadGen3D.git
cd RoadGen3D
git submodule update --init
```

### 2. 安装 Python 依赖
```bash
# 安装 M1 阶段依赖（基础张量/检索/测试）
.venv/bin/python -m pip install -r requirements-m1.txt

# 安装 M2 阶段依赖（网格处理/图像处理/GLTF）
.venv/bin/python -m pip install -r requirements-m2.txt

# 安装 UI 依赖（FastAPI/Uvicorn/HTTPX/Pydantic）
.venv/bin/python -m pip install -r requirements-ui.txt
```

### 3. 安装前端依赖
```bash
# 安装工作台前端依赖
make workbench-install

# 安装查看器前端依赖
make viewer-install
```

### 4. 下载 CLIP 模型（离线模式）
```bash
# 下载 CLIP 模型到本地目录
huggingface-cli download openai/clip-vit-base-patch32 \
  --local-dir models/clip-vit-base-patch32
```

## 第一个场景生成

### 命令行场景生成
```bash
.venv/bin/python scripts/m3_01_compose_street.py \
  --query "modern clean urban street" \
  --manifest data/real/real_assets_manifest.jsonl \
  --artifacts artifacts/real \
  --out-dir artifacts/real \
  --length-m 80 \
  --road-width-m 8 \
  --sidewalk-width-m 2.5 \
  --density 1.0 \
  --seed 42 \
  --design-rule-profile balanced_complete_street_v1 \
  --model-dir models/clip-vit-base-patch32 \
  --local-files-only \
  --export-format both
```

**输出文件**：
- `artifacts/real/scene.glb` - 3D 场景网格文件
- `artifacts/real/scene_layout.json` - 场景布局信息

### 关键参数说明
- `--query`：场景级文本查询
- `--length-m`：道路长度（米）
- `--road-width-m`：道路宽度（米）
- `--sidewalk-width-m`：人行道宽度（米）
- `--density`：资产密度
- `--design-rule-profile`：设计规则配置文件
- `--export-format`：导出格式（glb/ply/both）

## Web 工作台与查看器

### 工作台操作流程
1. 打开工作台：`http://127.0.0.1:4174`
2. 在对话面板中描述设计目标（例如："步行安全，全龄友好"）
3. 系统从知识库中检索相关设计指导和证据
4. 查看生成的设计草稿 - 可编辑参数与引用标注
5. 确认后提交场景生成作业
6. 在查看器中探索 3D 结果：`http://127.0.0.1:4173`

### 支持的布局模式
- **Graph Template**：预定义街道图（如 HKUST 广州校区入口）
- **OSM**：从 OpenStreetMap 提取真实街道（指定边界框）
- **MetaUrban**：基于街区的参考计划
- **Template**：简单参数化直线街道

### API 端点概览
- **设计相关**：`POST /api/design/draft`、`POST /api/design/generate`
- **场景作业**：`POST /api/scene/jobs`、`GET /api/scene/jobs`、`GET /api/scene/jobs/{job_id}`
- **知识检索**：`GET /api/scenes/recent`、`GET /api/knowledge/sources`、`POST /api/knowledge/search`
- **模板资源**：`GET /api/graph-templates`、`GET /api/reference-plans`

## 常见问题排查

### 环境自检
使用环境检查脚本生成报告，确认 Python 版本、平台、包安装状态与 CUDA/MPS 可用性：
```bash
.venv/bin/python scripts/m1_00_check_env.py --out artifacts/m1/env_report.json
```

### 模型下载与离线使用
如果网络受限，可以使用离线模式：
- 将 CLIP 模型下载到 `models/clip-vit-base-patch32` 目录
- 运行时添加 `--local-files-only` 参数
- 设置 `--model-dir` 指向本地模型目录

- API 服务：8010 端口

如果端口被占用，系统会自动提示或选择可用端口。

- **M1 依赖**：基础张量/检索/测试

- **UI 依赖**：FastAPI/Uvicorn/HTTPX/Pydantic

## 操作系统注意事项

### macOS（Apple Silicon）
- 注意 MPS（Metal Performance Shaders）可用性
- 如果没有 GPU，CPU 仍可正常运行
- 推荐使用 Rosetta 兼容模式运行某些依赖

### Linux
- 确保系统具备必要的编译依赖
- 如果 faiss 编译失败，可尝试使用系统包管理器安装：
  ```bash
  # Ubuntu/Debian
  sudo apt-get install libfaiss-dev
  
  # CentOS/RHEL/Fedora
  sudo yum install faiss-devel
  ```

### Windows
- 建议使用 WSL2（Windows Subsystem for Linux 2）
- 如果出现路径/权限问题，请以管理员身份运行终端
- Python 路径中的空格可能导致问题，建议使用标准安装路径
